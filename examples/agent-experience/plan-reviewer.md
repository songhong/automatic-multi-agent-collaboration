# plan-reviewer Experience Library

## Purpose
Reusable lessons for reviewing planning quality, task package completeness, question readiness, and shallow-plan prevention.

## Entries

### Plan Depth Gate
- SOURCE_AGENT: plan-reviewer
- TASK_ID: GLOBAL-SEED
- FAILURE_TYPE: shallow_plan_review
- PRINCIPLE: A plan should be rejected when it is only a module list; implementation plans must describe user success criteria, material mapping, architecture or data flow, module-level execution details, dependencies, validation strategy, and deliverables.
- WHY: Developers execute from the plan and task packages; shallow bullets force them to guess and produce low-quality work.
- DO_NOT_REPEAT: Do not pass plans that only name files or modules without explaining how and why they will be changed.
- LAST_REFERENCED: never
- REFERENCE_COUNT: 0
